Register

Posted on 2017-10-29 | In Computer Program , CPU , Register

Points:

  1. What is Program: A collection of instructions that guid computer(CPU) to run per step
  2. What’s inside the Program: Data and instructions
  3. What’s machine language: CPU knows and could directly run

The program regard register as object. From prespective of software
programmer: CPU is a collection of Register.

The register could store data and instruction, while the data could also
divide as arithmetic value and address.

Different register store different kinds of data. 

1. Calculation data ----> accumulator register.
2. Memory address ----> base register and index register.
3. Program counter ----> store the address of next instruction refered
   by conrol unit.
4. Flag register ----> the CPU status after running calculation or
   process.
5. general purpose register: include any data(multiple one)
6. instruction register: Only used by CPU internaly 
7. stack rigister: record program initial address

Program running Mechanism:
First mechanism:
Program counter detmerine: the flow of program. When user start program, the
operating system copy the disk content to memory. Using address to seperate
the instruction and data in meomry. Suppose the address 0100 is the
beginning address of program, once loading program to memory, the program
counter would start as 0100. Once CPU execute per instruction, the program
counter would increment by 1(This is sequential execution), then control
unit in CPU would read new address 0101 to fetch instruction and execute.

Second mechanism:
Except the sequential execution, there are also condition execution and loop
execution. For condition execution, the Flag register record the current
running status of calculation data, also responsible for overflow and parity
check.

Third mechanism:
When you call function using call and return method, using program counter and stack rigister.

When Memory refer to address: using base register and index register to
refer-> fulfill the functionality of index in advanced programming language.

CPU processing content:

  1. Data transfer instruction (read/write): register to memory, memory to memory, register to outer layer
    device(keyboard, mouse, monitor, router…)
  2. arithmetic instruction: calculation, logic processing, comparision and shift
  3. jump instruction: sequential execution (condition, loop)
  4. return/call instruction

Notes:
Assembly using memonic to program: each electronic signal corresponding to
memonic to represent

32 bits machine means there are 32 bits length in register, so data or
address access would be limited by 0 - 2 ^32
